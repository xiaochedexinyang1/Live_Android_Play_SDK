1. 文档介绍

1.1 文档目的

利用 Android Live SDK 可以与 CC 视频云直播服务进行对接，在android端使用cc视频的直播功能。当前 Android Live SDK 的版本具有直播问答、聊天、互动(签到、抽奖、投票)、回放(在线回放、本地回放)功能。

1.2 术语和缩写解释

2. SDK使用介绍

2.1 依赖库

dwlivesdk.jar：cclive核心库，仅支持android studio

core.jar : demo中二维码扫描依赖库

libjingle_peerconnection.jar：连麦依赖库

2.2 连麦库

libjingle_peerconnection.jar ： cc连麦jar包，dwlivesdk.jar引入后需要导入此文件 libjingle_peerconnection_so.so ： cc连麦动态库，引入连麦jar包后需要导入此文件

2.3 播放库

libijkffmpeg.so、libijkplayer.so、libijksdl.so： 播放动态库，引入dwlivesdk.jar后导入项目中

2.4 混淆编译

dwlivesdk.jar已经混淆过，如果需要对应用进行混淆，需要在混淆的配置文件增加如下代码，以防止livesdk的二次混淆：

 
-keep public class com.bokecc.sdk.mobile.**{*;}
-keep public class tv.**{*;}
-keep public interface com.bokecc.sdk.mobile.**{*;}
-keep public class org.webrtc.**{*;}

2.5 AnroidStudio配置sdk说明

  1. 使用AndroidStudioLib文件下的dwlivesdk.jar
  2. 复制jniLibs包里面的文件到工程的jniLibs文件夹下
  3. 配置依赖
 
compile ('io.socket:socket.io-client:0.7.0') {
 // excluding org.json which is provided by Android
 exclude group: 'org.json', module: 'json'
}

3.4 备注与说明

  1. 打开调试log的方法如下:
 
HttpUtil.LOG_LEVEL = HttpUtil.HttpLogLevel.DETAIL;
  1. 直播产品不建议使用暂停功能；
  2. 问答功能需要单独做一个数据结构存储，因为重新加载直播时，全部的历史问答信息会重新推送；
  3. 建议把DWLive初始化和stop的方法放到播放activity的生命周期里，这样可以防止出现打断情况而导致无法播放的问题,如下：
 
……
@Override
protected void onPause() {
super.onPause();
dwLive.stop();
}
@Override
protected void onResume() {
super.onResume();
dwLive.start(holder);
}
​
@Override
protected void onDestroy() {
dwLive.onDestroy();
super.onDestroy();
}

6. SDK日志输出功能

SDK提供了日志输出工具类：LogHelper。主要功能就是：将直播过程中相关的日志生成文件，存储在文件系统，用于调用分析。 使用方法：

  1. 在AndroidManifest.xml里面声明权限：
 
<uses-permission android:name="android.permission.WRITE_EXTERNAL_STORAGE"/>

并在适当的时机动态申请此权限。

  1. 在Application类里面执行
 
// 初始化日志记录模块
LogHelper.getInstance().init(context, true, null);

第一个参数是应用的上下文Context。 第二个参数是是否输出日志的开关。 第三个参数是自定义目录，此参数会影响日志输出的目录。如果第三个参数不为空，则日志目录为：

 
Environment.getExternalStorageDirectory().getAbsolutePath() + File.separator + dir + File.separator + "cclivelog"
  1. 在需要输出日志的时候执行，这个在SDK内部相关方法里面做了输出，用户也可以自行在App中输出，具体方法为：
 
LogHelper.getInstance().writeLog("*****");

  1. 建议仅在测试过程中使用此模块功能。
